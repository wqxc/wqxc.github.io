API -- createRef

createRef

创建一个通过ref属性附加到React元素的ref

只能使用在 HTML元素 或者是 类组件中，函数组件中无效

背景

DOM时代，使用document.getElementById 来获取DOM，进而操作

但是在虚拟DOM时代，这样的方式是行不通的，所以，虚拟DOM都会提供一个ref属性来绑定真实DOM

使用

像下边这样拿到DOM

ref的值

1、ref用在html元素上的时候，current属性接收底层的DOM作为其值

2、ref用在类组件上的话，current 的值是 类组件的实例,因为拿到的是类组件的实例，所以可以调用实例里的方法，
如此便可以在父组件上操控子组件的方法

3、函数组件没有实例，所以也不会有ref

ref属性更新的时机

在componentDidMount之前，或者componentDidUpdate 触发之前更新ref属性的值

ref回调函数

React 将在组件挂载时，会调用 ref 回调函数并传入 DOM 元素，当卸载时调用它并传入 null。在 componentDidMount 或 componentDidUpdate 触发前，React 会保证 refs 一定是最新的

ref 属性接收一个回调函数，来讲DOM元素存储在组件的变量中

回调形式的ref，可以在组件之间传递

ref转发

在父组件中为子组件添加ref可以访问子组件的实例，但是，这样就打破了组件的封装原则，耦合加深，
所以可以使用ref转发来做到这样的事情。

ref转发就是允许组件接收ref，并将其向下转发给子组件。

getDerivedStateFromProps

getDerivedStateFromProps

作用：让组件在 props 变化时更新 state。

但是，直接复制props给state是不可以的，state的来源应该尽可能的单一，
所谓的单一，也就是 state要么是组件自身控制的，或者是完全来源于props，而不需要自身来控制，就像是受控组件那样

所以说这个函数的用处是很小的。

这个hook更像是 vue的watch，
父组件更新了，或者是props改变了，这个方法就会被调用。

所在组件的state更新了也会调用

可以在这个方法中拿到props来更新子组件的state。

使用准则

此方法应该适用于state用于取决于props的组件

一个例子：

getDerivedStateFromProps 派生了三个state：prevPropsList、prevFilterText、filteredList

这个组件在getDerivedStateFromProps 中过滤了一下，避免重复的计算 只有当 props.list 或者是prevFilterText 发生变化了之后，才会重新计算。

这个模式避免了重复了计算，但是，太过于复杂，需要追踪 props和state的变化

可以使用下边这样的方式来简化

每次render（也就是 state更新或者是props更新）之后都会调用，然后重新渲染，但是，这样就会出现一些不需要的渲染，
比如，父组件的state发生了变化，但是，props.list 和 filterText 都 没有变化的时候，渲染是多余的。
因为是 继承 了PureComponent 所以，过过滤掉一些多余的渲染，但是，别的props变化的时候，还是会触发渲染的。
在过滤很大的列表的时候，效率不好

React提供了 memoization 函数来帮助不必要的过滤

componentDidUpdate

接受三个参数：

preProps: 更新之前的props，可以用来和当前的props(this.props)对比
prevState: 更新之前的state，可以用来和当前的state（this.state）对比
snapshot: 组件实现了 getSnapshotBeforeUpdate 方法的话，其返回值为 该参数，否则的话就是undefined

3、如果shouldComponentUpdate返回了false，这个函数就不会执行

React

componentdidmount

componentdidmount

1、调用的时机是: render之后，组件被渲染到DOM中之后运行

2、一般会在这个生命函数里进行ajax请求

3、在这里添加订阅，在销毁的时候取消订阅

4、可以在这里直接调用setState，触发额外的渲染，但此渲染会发生在浏览器更新屏幕之前，
保证，即使有两次render，用户不会看到中间状态

5、不能依赖第四项，如果渲染依赖DOM节点的大或者是位置，可以使用
比如 modals tooltips 需要先判定要展示的位置，而后才展示

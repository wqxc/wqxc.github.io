逻辑重用

逻辑重用

代码复用是一个很大的主题，React 社区也一直有各种各样的代码复用的技术。

从 Mixins 到 Render Props 到 HOC 再到 Hook，对于 代码复用，我们一直在探索，

每一种的技术在当时来说都是比较优秀的解决方案，但是，随着时间的推移，这些方案也都出现了

或多或少的问题。

思想

所谓的逻辑重用的核心是，在不改变原有组件的前提下，抽离代码，并复用。

像是上边说的那几种逻辑重用的方式， 也都是在不修改原有的类型、对象、函数的前提下，为其添加新的属性、方法，以此来增强其功能。

较为学术的说法是 符合程序的设计原则： 开发封闭原则和 单一职责原则

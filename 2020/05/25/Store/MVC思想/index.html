流行的MVC

牛叉的思想 MVC

有人说 MVC 是一种框架，其实 MVC 更算是一种思想，或者叫做 设计典范 或者是 设计模式

MVC 的厉害之处就在于，将原本混做一团的东西，分为条理清晰的不同层次，不同的层次之间有管道相连，高内聚的前提下而又低耦合。

MVC

MVC 是比较普遍的前端框架设计模式。

框架的作用是为了代码的重用，而设计典范或者是设计模式是设计的重用。

一般来说 MVC 框架 核心分为三个部分。

Model

负责管理数据。大部分的业务都应该在 model 中进行，比如 ajax 请求。

View

也叫做视图部分，负责展示页面。也就是 Html，虽然，HTML 中可以进行事件以及数据的操作。但是，最好要避免在这里进行涉及业务逻辑的操作。

Controller

负责接受用户的输入。根据用户行为调用特定的 Model 部分逻辑。最后 Model 生成了数据，通过 Controller 交给 View。最终展现在用户面前。

MVC 的特点 – 前端 。

1、优势： MVC 框架很好的做到了分层的概念。不同的部分只做自己应该做的事情。相互独立，却又紧密联系。也就是高内聚，低耦合。分而治之，避免将逻辑和界面的渲染混杂在一起。

2、缺点：世界上本身就不存在完美的事务，对于 MVC 来说也是。MVC 框架只是希望开发人员遵守：不要进行 View 和 Model 进行通信。但是，View 和 Mode 通信的便利总是在不经意间诱惑开发人员去破坏这一条规则。

总结

如果能够完全按照 MVC 的思想来编写代码，依旧能够保证分层，代码逻辑清晰。但是，每新增一个业务就要新增 Controller 和 Model，这会导致项目臃肿。

因为 MVC 在前端有这些缺点，所以才有了抛开 MVC 的一些创造，于是也就有了前端的状态管理设计模式，基于此，也有了后来的很多状态管理框架。比如 Redux，Vuex 等。

而前端状态管理的设计模式便是 Flux。

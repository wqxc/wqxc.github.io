React 更新逻辑

setState

关键点:

setState不会立刻改变React组件中state的值；
setState通过引发一次组件的更新过程来引发重新绘制；
多次setState函数调用产生的效果会合并。

更新

不像是是vue监听响应式属性的变化来引发watcher，进而重新渲染。
React 则是依靠setState 这个API来引发一次组件的更新过程，进而渲染页面

多次调用setState 会引发批处理
isBatchingUpdate 会被设定为true

setState 引发的生命周期函数

shouldComponentUpdate
componentWillUpdate
render
componentDidUpdate

+++++++
顺带一说，props变化引发渲染，所调用的的生命周期函数比state多了一个：

componentWillReceiveProps
+++++++

shouldComponentUpdate return false 的时候 state就已经发生了变化

shouldComponentUpdate return true 的时候 state还没有变化
componentWillUpdate state也没有变化
render 这个阶段 state才更新了

所以说，我们setState的时候，是没法立刻拿到更新后的state的值得

setState 缺点

唯一的缺点就是 不像是字面上理解的那样理解修改state，而是延迟处理。

但是，依旧有方法解决这个问题。

那就是接受一个函数来作为参数

setState({count: count + 1})
setState({count: count + 1})
setState({count: count + 1})

同一时间调用三次，最终也只会执行一次 count +1

想要达到加三次的效果，就需要使用函数参数

function increment(state,props){
retun {count: state.count + 1}
}

setState(increment)

setState(increment)

就可以实现加两次。

另外的一种方式是使用 addeventListener 添加事件，

addeventListener 添加的事件，不会开启批处理，也就是不会合并state

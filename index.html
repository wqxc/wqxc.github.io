Effect-Hooks

Effect-Hooks

我们知道 React 提出 hook的概念主要是为了让无状态组件 进阶为 函数组件，以此获得像

类组件一样的能力，但是，却别类组件的性能要高。

State-hooks 主要是辅助 无状态组件 可以像类组件一样 保存自己的状态。

但是，类组件的另一个特点是 生命周期。

其实声明周期本质上也是 钩子函数。而且，生命周期中最重要的其实就三个：

componentDidMount

componentDidUpdate

componentWillUnmount

而 Effect-Hoos 就可以看做是这是三个声明周期函数的汇合版本。

Effect-Hoos 顾名思义就是 副作用钩子函数，主要是处理副作用。

副作用

什么是副作用呢？ 比如在组件中获取数据，修改DOM等都是副作用。

而React 中的副作用 一般分为两类，需要清除的和不需要清除的。

不需要清除的：

比如请求数据

手动变更DOM

记录日志等

需要清除的：

监听、订阅了外部的数据源

需要清除的副作用，如果不清除的话就有可能造成内存泄露等。

React►组件►1、函数组件和类组件

React 组件

React中的组件

React中的组件粗略的分类的话，有两种：

函数组件

类组件

函数组件

函数组件很简单，就是一个函数，如下：

函数组件的作用很简单，就是返回一个 DOM 出来，这个DOM的复杂度是不一定的，有时候很复杂，有时候很简单。但是，无论是简单还是复杂，与同样的 类组件相比，函数组件的性能是更优越的。

函数组件的特点

1、函数组件是没有 this这个对象的

2、函数组件没有生命周期

3、函数组件没有状态

函数组件的根源是一个返回 类DOM的函数。我们知道，函数被调用完成之后就会被回收。所以，函数组件不会有 状态可供操作。

另外的，函数组件的思想是无状态组件，函数组件没有生命周期，也无需去考虑什么情况下用什么钩子函数。这就是说，函数组件只是一个展示组件

这里说一下，所谓的展示组件，就是，并非是完全没有处理逻辑，    比如Ajax等还是要有的。

所谓的没有处理逻辑是不会有 比如 填写表单之类的交互逻辑

函数组件的用处

在最理想的情况下所有的组件都应该是函数组件，这样的应用性能是最佳的。函数组件有自己的特点，这也是我们在写组件的时候要首先考虑一下，是否要使用函数组件而不是一定要用类组件。

类组件

类组件就是使用 类的方式来构建组件,如下所示。

类组件的特点

1、有this

2、有生命周期

3、有状态

类组件的核心是一个类，这个类的核心是 render方法，render方法就是用来返回一个类DOM结构。

类组件在使用的时候首先要进行实例化，而这也是类组件相对于函数组件性能要稍微差劲一些的原因。

类组件的用处

类组件为我们定义了 生命周期方法，并且提供了 状态，使得我们可以在组件内部进行方便的逻辑交互。

比如，表单的交互

在我看来类组件的最大特点就是他可以保存状态，而函数组件无法保存状态。

为什么类组件可以保存状态而函数组件不行

实例化之后的类实例是一个对象 比如这样的 obj = {}

obj 对象里完全可以放置任何你想要的属性，

比如 类组件就是放置了  render函数，生命函数，state等

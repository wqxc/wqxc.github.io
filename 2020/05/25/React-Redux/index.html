React-Redux 对 Redux 的升华

React-redux 产生的背景

容器组件和傻瓜组件

背景

为什么要将组件分为容器最近和傻瓜组件呢？这个要从 Redux 框架下来看。

在 React 中使用 Redux 框架，一个 React 组件基本上就要完成以下两个功能。

1、和 Redux Store 打交道，读取 Store 的状态，同时要监听 Store 的状态改变，在 Store 发生改变的时候渲染组件，而在需要更新 Store 的时候，就要派发 action 对象。

2、根据 props 和 state 渲染用户界面

我们知道，组件的核心原则是，一个组件只做一件事。如果一个组件做的事情太多，就需要考虑拆分了。

而这也是，为什么我们要在这里拆分为容器组件和傻瓜组件。

容器组件

我们将承担 第一个功能的的组件称之为容器组件。也就是说，容器组件主要是负责和 Redux Store 打交道。

他主要处在最外层，所以叫做容器组件。

傻瓜组件

也就是容器组件内的组件，这个被容器组件包裹的组件叫做傻瓜组件，因为这个组件主要是根据容器组件给的数据进行无脑的展示就好，所以也叫做傻瓜组件。

解释

如果我们的项目中有多个组件都需要使用 Store，那么我们也都应该将其抽离为 容器组件和傻瓜组件的结合。

而多个容器组件和傻瓜组件的拆分会产生很多的相似代码，这些相似的代码其实还可以进一步的抽离。

比如一个容器组件，主要是 获取 store，并监听 store 里状态变动等等。这些其实都可以抽离出来以减少重复的代码的。

而这减少重复代码 就是 react-redux 库的拿手戏了。

useEffect

useEffect 也叫做 副作用钩子函数。顾名思义，他的作用就是用来执行一些副作用操作。

副作用

什么算是副作用呢？比如一些IO操作，HTTP请求等。

副作用和纯函数是一个相对的概念，纯函数是说返回值依赖于他的参数，给定相同的参数，必然返回相等的返回值。
而如何函数内部做了和运算返回值没有关系的事情，比如修改了全局变量，修改了传入的参数，发送请求，console，手动修改DOM等都是副作用

类比类组件

useEffect Hook 可以看做是 componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个函数的组合。

useEffect 会在DOM更新之后调用，也就是 组件挂载 和 更新阶段执行，也就是 componentDidMount，componentDidUpdate 两个类的生命周期方法中。

其实我们无需去考虑生命周期，而只需要记住，useEffect 会在DOM更新之后调用，而无需在意它是mount还是update

useEffect会返回一个函数，这个函数会在下一次 useEffect执行的时候执行。

渲染与帧

帧的概念

参考文章：[https://musicfe.dev/react-hooks/]

我们可以认为每一次组件的渲染被认为是一帧

每一帧都拥有自己的状态

当然，这个状态在函数组件中表现的更为明显，举个例子：

我们定义了一个组件 Example2,考虑一下先点击一下 Delay setCount 按钮，而后在三秒内连续点击三次setCount按钮，最终 count 的值会是多少呢？
直观上来看count应该是 4。但是，事实上 count应该是1。

步骤如下：

1、点击 Delay setCount ，setTimeout函数等待执行
2、点击了三次setCount (当然，这是在seTimeout等待执行的三秒内点击的三次)，count变为3
3、seTimeout 定义的回调函数执行了，此时的核心是 回调函数里的count 是多少？

考虑一下，我们在点击 Delay setCount 按钮的时候，count是多少呢？ 此时是 0，应该是毫无疑问的。
也就是 三秒之后 执行 setCount(0 + 1);

接下来是点击了三次 setCount 按钮之后，count 变成了3

此时的count并不会影响setTimeout函数内的count值，这就是React中帧的概念，每一帧的状态都是没有一点关系的独立个体。

详细的分析一下

Delay setCount 点击的时候做了什么？
首先是 handleClick 被调用了 ，count是多少呢？我们可以认为是下边这样的。

接下来，第一次点击 setCount 按钮, count 被更新了 函数重新执行，以渲染。
count 变成了

接下来是第二次，第三次，函数依旧会重新执行，count 就变成了 3

最后，setTimeout的回调函数到期开始执行，执行的方法当然是

count 又恢复到了 1

类组件中的表现

函数组件 或者是叫做 hook组件，我们应该将其和类组件完全的分割开来，hook本身不是用来代替类组件的。同样的例子，我们看一下在类组件中的表现

调用一次Alert Count 按钮，而后点击 三次 setCount ，最后alert 的 count值是 4.

总结

为什么出现上边这样的差距呢？核心是 在于 this。

类组件有自己的实例，每一次的修改是在实例上修改的，this永远指向的是最新的实例，最新实例拥有最新的state。

函数组件没有自己的实例，每次渲染都是执行一次函数，函数执行完毕之后就被销毁了，每一次渲染，
虽然函数组件内的函数，属性都看着和原来的一样，但是，实际上他们已经被销毁之后又再次创建了。

当然，函数组件可以使用 useRef来实现跨组件的保存状态

ref 属性接收一个回调函数，来讲DOM元素存储在组件的变量中

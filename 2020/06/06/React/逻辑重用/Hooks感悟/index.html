React-Hooks

一些感悟

我们知道，函数组件其实就是 一个函数，函数不像是 类实例那样可以保存下来，无需重复的实例化，而
函数组件每一次的加载都是重新执行了一遍函数，所以函数内的 hook 其实也是重新被执行了，

只是，React 在内部的机制中将其保存了一下，

自定义 Hooks

所谓的 Hooks 完全就可以被认为是一个函数，我们完全可以通过自定义 hooks 的方式抽离公共的内容。

比如两个以至多个组件都会用到登录状态。

我们将登录的状态抽离出来为 useLoginStatus

而后，在使用 状态的组件中引入。

如上所示，useLoginStatus 是我们自定义的 hook，其内一个状态 isLogin，这个状态被 uploadSecretFile，

downloadSecretFile 两个组件需要，而 isLogin 的 值需要通过某种逻辑获取到，比如，请求后台等等。

而 获取是否登录的逻辑 两个组件都无需去关心，只需要拿到最终的值就行了，如此经过我们自定义的 hook 之后，

状态就被抽离了。如此一来，不就实现了代码的重用，逻辑的重用，状态的重用了。

自定义 Hook 本身就是一个函数而已

如果，没有 hook 我们想要共享这些 状态逻辑的话，就只能使用 HOC（高阶组件） Mixin 或者 Render Props 几种方式。

但是这几种方式都没有 hook 优越。

React-Hooks

React Hooks

为什么要有React Hooks

以往，出现了一门新技术，总想着拿来就开始尝鲜，但是，却很少去考虑，为什么要有这一门技术，以及，这一门技术是怎样的？

后来慢慢的发现，开始接受一项新的技术的时候，第一个要考虑的不是怎么使用这个技术，而是，为什么会有这个技术？

简单的来说就是： 我们应该知道这个技术出现解决了什么痛点。

痛点

1、越来越难以复用。

2、组件越来越复杂。

越来越难以复用

React中复用状态逻辑的解决方案一般是：render props 和高阶组件（而关于 Render Props 和 高阶组件，我们会另外说）。

难以复用的原因是组件的内部状态： state。

我们会经常在组件中使用 state，而state是针对该组件本身的，复用的可能就打折扣了。

我们都知道 一个无状态组件是最好复用的。给什么就渲染什么。但是，有状态的组件就是因为 state的存在而使得复用的可能性下降。

内部state越多，复用的可能性就越小。

组件越来越复杂

组件中有不同的生命周期，在不同的生命周期中做不同的事情，比如在

construct 方法里 发起 ajax 请求等。

不同的生命周期里做不同的事情。

甚至有时候一个操作逻辑会在不同的生命周期函数中。

比如：
初始化的时候监听一个方法

数据变动的时候调用方法

组件销毁的时候清楚监听方法

更新的时候判断是否要更新

如此一来，一个组件就越来越大，越来越臃肿，而且很难对其拆分。

Hooks 背景

除开 以上的两个 问题之外，让人难以理解的 class也是一大弊端，你需要理解 this的绑定，使用bind方法来绑定。这都是不慎友好的部分。

而 Hooks 的 提出则是基于这些问题的 一个解决方法。

hooks搭配 函数组件就以完成 与类组件相似的作用。

使用自定义的 hooks 就可以提取组件的公共部分，使得组件的复用更简单。

并且组件不需要再使用生命周期造成混乱复杂的逻辑。

Hooks 的作用

1、为共享状态逻辑提供了原生途径。--自定义hooks

2、组件关联部分拆分--Effect Hook

3、使得函数组件可以使用更多的React 特性

什么是 Hooks

Hook 其实就是一个比较特殊的函数

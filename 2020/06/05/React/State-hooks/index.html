State-hooks

我们知道 函数组件无法保存状态，所以在函数组件中无法使用状态。

但是，在State Hooks加持下，函数组件就完全可以有自己的状态去操做。

显示引入 Hooks 也就是 useState

这里说一下 useState 这个hooks 主要是为 函数组件提供 state，让函数组件可以有状态可以操作。

主要的使用方式是

第一是 引入

第二是 定义

定义的时候是 使用数组的解构。 数组的第一项是  定义的state 变量 比如 count。等价于 类组件里的 this.state.count.

第二个参数  setCount 是用来改动 变量的 类似于 类组件的 setState方法。

就如上所示的那样，我们定义了一个 变量 count 并其初始值为0，而且提供了一个修改这个变量的方法 setCount.

如此一来， 函数 Example 执行完成之后，依旧保留了变量了 count，与操作 count 的方法。这就和类组件一样了。

因为有了 hooks的加入，使得 无状态组件 转化为了函数组件。

这里，我们将 不使用 hooks的函数组件叫做 无状态组件。

而将，使用了hooks的无状态组件叫做 函数组件。 这是一种意义上的称呼。
